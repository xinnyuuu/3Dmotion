# 相机分辨率与帧率选择记录

这份文档记录当前项目如何从现有相机支持的分辨率、帧率和格式中选择主线采集档位。结论不是简单选择最高分辨率，而是结合项目目标、视场比例、运动追踪稳定性、同步压力、存储压力和后处理链路共同决定。

## 当前项目需求

3DMotion / EgoHand 当前要优先解决的是：

- 头环相机 + 头环 IMU 输出稳定的 `T_W_H`。
- 头环相机观测手环 AprilTag，输出稳定的 `T_H_B`。
- 后续把 `T_W_H` 和 `T_H_B` 组合成手腕 / 手部运动轨迹，服务 WAM、VLM、遥操和机械臂动作迁移。

因此相机配置的优先级是：

1. 时间连续性和帧间运动可跟踪。
2. 画面比例尽量保留手部、桌面、胸前操作区和左右环境。
3. AprilTag、手部和物体边缘足够清晰。
4. 多路相机可以稳定采集，不频繁丢帧。
5. 离线 AprilTag / OpenVINS / rosbag2 处理的数据量可控。

当前工程路线不是直接做实时闭环，而是：

```text
dashboard 录制 raw session
-> validate_session / session_quality 检查
-> 离线 AprilTag 得到 T_H_B
-> 离线 OpenVINS / rosbag2 replay 得到 T_W_H
-> 离线融合、评估、导出 WAM motion stream
```

所以分辨率和帧率的选择首先服务于“录下可反复处理、可标定、可验证的数据”，而不是服务于在线实时推理的最低延迟。

## 选择结论

最终不要用一个档位覆盖所有任务，而是按用途分档：

| 用途 | 推荐配置 | 原因 |
| --- | --- | --- |
| 当前离线处理主线 session | `MJPG, 1600x1200 @ 25fps` | 当前四台相机都能覆盖的高质量 4:3 档位，适合录制、标定、离线 AprilTag 和 OpenVINS 调试。 |
| 单路 / 双路高动态实验 | `LRCP imx415, MJPG, 1280x960 @ 60fps` | 4:3 无明显上下裁剪，60fps 更适合测试快速头动、手腕动作和时间对齐敏感性。 |
| VLM 高质量第一视角视频 | `LRCP USB3.0, MJPG, 1920x1080 @ 60fps` | 1080P 60fps 是通用视频模型最友好的输入档位，画质和时间连续性都好。 |
| 高精度静态标定 / 检查镜头质量 | `2592x1944 @ 25/30fps` | 5MP 4:3 适合拍静态标定板和检查细节，但不适合默认追踪。 |
| 多路压力测试 / 低负载回归 | `800x600 @ 60fps` 或 `640x480 @ 60fps` | 数据吞吐低，适合检查采集稳定性、USB 带宽和多路同步。 |

当前最重要的主线判断是：

```text
离线主线采集优先选择稳定的 4:3 高质量共同档位，而不是最高分辨率或最高帧率。
```

如果只能在当前 dashboard 中用统一 profile 录四路，推荐先用：

```text
MJPG 1600x1200 @ 25fps
```

如果要专门测试更高动态、更密集视觉观测或时间对齐敏感性，再单独用 imx415 测：

```text
MJPG 1280x960 @ 60fps
```

## 目标头环硬件规格

当前相机选择是为了让项目先跑通采集、VIO、AprilTag wrist tracking 和后处理链路。长期目标仍然是定制头环上的大视场鱼眼相机，而不是普通 USB 相机的矩形画幅。

目标头环方案里，相机分为两类：

| 相机角色 | 推荐规格 | 主要用途 |
| --- | --- | --- |
| 第一视角主摄 | `1.4mm fisheye, 220-240 deg diagonal FOV, 2580x2580, 1:1 square sensor, 15-30fps` | 负责正前方 hand-object interaction、action segmentation、细粒度动作和关键物体细节。 |
| 头环补盲 / 环境覆盖相机 | `1.4mm fisheye, 220-240 deg diagonal FOV, 2Kx2K, 1:1 square sensor, 15-30fps` | 负责左右补盲、环境上下文、空间覆盖和头部转动时的连续背景。 |

这和当前 USB 相机的选择并不矛盾。当前阶段用 `4:3 + 25fps` 作为离线主线档、用 `4:3 + 60fps` 作为高动态实验档，是在现有相机能力下的工程折中；目标硬件追求的是：

```text
更大 FOV
1:1 方形 sensor
固定刚性头环结构
多相机统一标定
可控的软件同步，后续升级硬件同步
```

图里的 15-30fps 目标主要面向第一视角数据采集、VLM 训练和人类动作理解，也符合当前“录制后离线处理”的原型路线。如果任务切到更高动态的 VIO、wrist 6DoF 或快速运动追踪，再额外验证 60fps 档位是否带来更稳定的视觉观测。

## 为什么不默认选择最高分辨率

`3840x2160`、`2592x1944` 这类档位对画质和静态标定很有价值，但不适合作为默认离线主线档位，原因是：

- 高分辨率会增加 USB 传输、JPEG 解码、磁盘写入和 rosbag2 体积压力。
- 当前 OpenVINS rosbag2 导出会把 JPEG 解码成 ROS `sensor_msgs/Image` 的 `bgr8` 原始图像，1080p 以上很容易变成数 GB 级别。
- 后续 VIO / AprilTag 处理更怕运动模糊、帧间位移过大、时间戳 jitter 和丢帧，而不是单帧像素不够。
- 高分辨率低帧率对快速头动、手腕动作和 AprilTag 角点跟踪不一定更稳。
- 相机和 IMU 还没有硬件同步时，帧率越低，单帧时间间隔越大，运动状态插值和时间偏移误差更明显。

所以主线录制先保证：

```text
稳定采集 > 标定一致性 > 合理视场比例 > 足够清晰 > 可离线反复处理 > 极限分辨率
```

## 为什么优先 4:3

项目关心的是第一视角下的手、桌面、物体、胸前操作区和部分左右环境。16:9 视频适合普通观看和 VLM 输入，但对头戴追踪有两个问题：

- 上下视野被裁剪，手部和桌面区域更容易丢失。
- 对鱼眼或大视场镜头来说，16:9 不能充分利用圆形成像区域。

4:3 比 16:9 更适合当前阶段的离线处理和标定，因为它在垂直方向保留更多内容，也更接近当前项目文档中的 4:3 sensor input 路线。

## 60fps 的位置

VLM、普通第一视角数据和当前离线原型主线可以先接受 15-30fps。VIO、AprilTag wrist tracking 和快速手部动作更依赖时间连续性，所以 60fps 仍然是重要的实验档。它的优势是：

- 帧间位移更小，特征跟踪和 AprilTag 角点检测更稳定。
- 快速头动时更不容易出现大跨度运动模糊和跟踪失败。
- 视觉轨迹和 IMU gyro / accel 峰值更容易对齐。
- 后续做 ESKF 或 smoother 时，视觉 correction 更密集。

因此：

```text
离线主线采集：优先 15-30fps 的稳定高质量档位。
高动态敏感性测试：再用 60fps 档位做 A/B 对照。
```

## 为什么优先 MJPG

两类相机都支持 MJPG。对于高分辨率和多路采集，MJPG 比 YUYV 更适合作为默认格式：

- MJPG 在 USB 上传输的是压缩图像，带宽压力更低。
- imx415 的 YUYV 高分辨率帧率很低，例如 4K YUYV 只有 1fps，不能用于追踪。
- dashboard 当前保存 JPEG 图片，MJPG 路线更贴近实际落盘格式。

YUYV 可以作为低分辨率调试或画质对照，但不建议作为多路高分辨率默认格式。

## 当前四路组合下的实际取舍

当前相机组合里同时存在 `LRCP USB3.0` 和 `LRCP imx415`。如果软件层要求所有相机使用同一个 width、height、fps，则要看四台相机共同支持的交集。

比较重要的交集档位：

| 交集档位 | 评价 |
| --- | --- |
| `1920x1080 @ 60fps` | 共同支持，帧率高，但 16:9 会裁掉上下视野。适合 VLM / 通用视频，不是追踪最优。 |
| `1600x1200 @ 25fps` | 共同支持，高质量 4:3，适合四路统一 session 和离线 AprilTag，但动态追踪帧率偏低。 |
| `1280x720 @ 60fps` | 共同支持，高帧率低负载，但 16:9 裁剪明显。 |
| `800x600 @ 60fps` | 共同支持，4:3 高帧率，适合压测和低负载追踪。 |
| `640x480 @ 60fps` | 共同支持，最小负载，但细节不足。 |

所以当前阶段建议：

- 需要四路统一、高质量 4:3 离线主线 session：选 `1600x1200 @ 25fps`。
- 需要四路统一、高帧率同步 / 带宽压测：选 `800x600 @ 60fps`。
- 需要单路 / 双路 imx415 高动态 A/B 实验：选 `1280x960 @ 60fps`。
- 需要给 VLM 导出普通视频：选 `1920x1080 @ 60fps`。

## 标定要求

相机内参必须和实际采集分辨率一致。切换分辨率后不能直接复用旧内参，尤其是从 16:9 切到 4:3、从 4:3 切到 1:1、或者从高分辨率切到低分辨率时。

每个主线档位至少要分别标定：

- `LRCP imx415 1600x1200`
- `LRCP USB3.0 1600x1200`
- `LRCP USB3.0 1920x1080`
- `LRCP imx415 1280x960`，用于高动态 60fps 对照实验

如果后续固定最终硬件和镜头，只保留最终采集档位对应的内参即可。

## 验证计划

最终档位不只靠规格表决定，还需要用真实 session 验证：

1. 每个候选档位录制 20-30 秒。
2. 检查 `frames.jsonl` 中 frame interval 是否稳定。
3. 检查四路 `skew_us` 是否可接受。
4. 检查是否有 capture error、丢帧、JPEG 写入失败。
5. 用 AprilTag 离线处理看 tag detection rate、reprojection error 和 pose jitter。
6. 用 `C0 + head_imu` 跑 OpenVINS readiness 和 rosbag2 replay。
7. 比较处理耗时、rosbag2 体积和轨迹稳定性。

推荐用下面几个候选档位做 A/B 测试：

| 候选档位 | 目的 |
| --- | --- |
| `MJPG 1600x1200 @ 25fps` | 四路统一高质量 4:3 离线主线候选。 |
| `imx415 MJPG 1280x960 @ 60fps` | 高动态和时间对齐敏感性对照候选。 |
| `MJPG 1920x1080 @ 60fps` | VLM / 通用视频候选。 |
| `MJPG 800x600 @ 60fps` | 多路低负载追踪和同步压测候选。 |

## 最终记录

当前阶段最终记录如下：

```text
Offline main recording profile:
  Camera: LRCP USB3.0 + LRCP imx415 mixed setup
  Format: MJPG
  Resolution: 1600x1200
  FPS: 25
  Reason: current high-quality 4:3 intersection for all available cameras; suitable for repeated offline AprilTag/OpenVINS processing.

High-dynamic comparison profile:
  Camera: LRCP imx415
  Format: MJPG
  Resolution: 1280x960
  FPS: 60
  Reason: 4:3 high-frame-rate profile for testing fast motion and timestamp sensitivity.

VLM video profile:
  Camera: LRCP USB3.0 or imx415
  Format: MJPG
  Resolution: 1920x1080
  FPS: 60
  Reason: standard 1080P high-frame-rate video input.
```

后续如果更换为支持硬件同步、global shutter、统一 1:1 鱼眼 sensor 的头环相机，应该重新评估最终档位。届时优先目标会变成：

```text
1:1 or 4:3 large-FOV sensor
30-60fps
hardware trigger / hardware timestamp
stable multi-camera calibration
```
