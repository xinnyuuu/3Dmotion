# 手部传感器位置需求与测试方案

这份文档用于给机械/硬件团队反馈手部摄像头和 IMU 的安装需求。目标不是一次性定死最终量产结构，而是用现有爪夹雏形和摄像头模块，把候选位置用统一方法测清楚，再输出可执行的结构修改建议。

## 1. 结论先行

本轮需要同时覆盖两种使用场景：

| 场景 | 第一目标 | VIO 要求 | 推荐传感器策略 |
| --- | --- | --- | --- |
| 穿戴爪夹 | 捕捉手指、虎口、爪夹末端和接触区域 | 必须评估，不能默认成立 | 虎口上方摄像头作为首选候选；若要声明支持 VIO，必须让画面保留稳定环境纹理，并把 IMU 与摄像头刚性固定 |
| 普通精细手部动作 | 捕捉指尖、掌心、接触点和细粒度手势 | 可选评估，不作为第一版硬约束 | 掌心内侧靠手腕方向和小指侧各测一组；优先看手部可见性和佩戴干涉 |

关键判断：

- 单个近距离手部摄像头不一定能同时做好手指特写和 VIO。VIO 需要足够静态环境纹理，画面不能主要由手指、爪夹和被操作物体占满。
- 只要文档或对外接口写“支持 VIO”，IMU 就不是可选件；它必须和 VIO 摄像头所在模块组成同一刚体，并满足时间同步和外参标定条件。
- 如果某个候选位手部可见性很好但 VIO 很差，应把它定义为“手部观察位”，不要让机械团队误以为它天然满足 VIO。

## 2. 传感器功能分层

### 2.1 手部观察摄像头

用于捕捉手指动作和手物交互，优先级如下：

1. 指尖、关键关节、虎口或掌心可见。
2. 爪夹场景下能看到爪夹末端、开合状态和夹取接触点。
3. 普通精细动作下能看到捏取、按压、旋拧、拿薄片等动作。
4. 画面不过度失焦、过曝、运动模糊或被手指遮挡。
5. 模块不影响抓握、夹取、转腕和接触物体。

### 2.2 VIO 摄像头

用于估计相机/手部刚体在空间中的运动，优先级如下：

1. 画面中必须长期保留足够静态环境纹理。
2. 快速手部动作时不应因为近距离手指占满画面而丢失视觉特征。
3. 摄像头和 IMU 必须相对刚性固定。
4. 相机内参、畸变、camera-to-IMU 外参和时间偏移必须可标定。
5. 若是 rolling shutter 相机，需要记录 readout time；优先使用 global shutter。

### 2.3 IMU

IMU 有两类角色，不能混用：

| 角色 | 可放置位置 | 硬性要求 |
| --- | --- | --- |
| VIO IMU | 与 VIO 摄像头同一刚体模块、爪夹基座刚体、手背刚性支架 | gyro + accel，建议 >= 200 Hz；相机-IMU 时间误差目标 < 1-2 ms；外参可标定 |
| 手部动作平滑 IMU | 腕部刚性手环、爪夹基座、手背刚性件 | 可用于 ESKF 或动作状态辅助，但如果不和摄像头刚性连接，不能直接当作 camera-IMU VIO 输入 |

禁止项：

- 不要把 VIO IMU 贴在会滑动的软绑带、皮肤、布料或线缆上。
- 不要让 IMU 和摄像头之间存在可见晃动、弹性形变或拆装后不可复现的相对位姿。
- 不要只记录 PC 接收时间后宣称相机-IMU 已同步；原型可先记录 host timestamp，但最终硬件需求应保留硬件时间戳或触发能力。

## 3. 候选安装位

### 3.1 穿戴爪夹场景

基准位置：摄像头在虎口上方，固定在爪夹或手背刚体结构上，朝向指尖和爪夹工作区。

测试高度：

```text
40 / 60 / 80 / 100 / 120 mm
```

测试俯仰角：

```text
35 / 45 / 55 deg
```

推荐候选编号：

```text
CLAW-H040-P35
CLAW-H040-P45
...
CLAW-H120-P55
```

每个候选位必须记录：

- 高度：摄像头光心到虎口参考平面的距离。
- 俯仰角：光轴相对虎口平面朝指尖方向下看的角度。
- 横向偏移：相对虎口中心向拇指侧/食指侧的偏移。
- 是否能刚性安装 IMU。
- 是否挡手、挡爪夹、挡物体接触。

首选倾向：

- 如果 `60-100 mm` 高度可以同时看到指尖、爪夹末端和部分静态背景，优先在这个区间内收敛。
- `40 mm` 容易获得手部特写，但有失焦、遮挡、VIO 背景不足风险。
- `120 mm` 更容易看到环境背景，但要重点检查模块高度、碰撞和线缆干涉。

### 3.2 普通精细手部动作场景

候选 A：掌心内侧靠手腕方向，朝向手指和掌心。

候选 B：掌心内侧小指侧，基本与小指方向平行，朝向指尖和接触区。

测试高度：

```text
30 / 50 / 70 mm
```

推荐候选编号：

```text
FINE-A-H030
FINE-A-H050
FINE-A-H070
FINE-B-H030
FINE-B-H050
FINE-B-H070
```

第一版评价重点：

- 是否能看到指尖和接触点。
- 是否影响五指自然弯曲、捏取、按压和旋拧。
- 是否容易被手掌、袖口或物体遮挡。
- 是否保留足够背景纹理。如果没有，则只定义为手部观察位，不写成 VIO 位。

## 4. 采集流程

每个候选位置至少录 3 段，每段 20-30 秒。

动作顺序固定如下：

1. 静止 3 秒，用于初始化和观察抖动。
2. 慢速开合爪夹或五指开合。
3. 快速开合、捏取、旋转手腕、靠近和远离物体。
4. 真实任务动作：抓杯子、夹取小物体、按按钮、旋拧、拿纸片。
5. 结束前静止 2 秒，观察回到静止后的漂移和抖动。

数据命名建议：

```text
scenario_position_repeat
CLAW-H080-P45_R1
FINE-B-H050_R2
```

3DMotion 采集方式：

```bash
cd ~/lxy/3DMotion
source .venv/bin/activate
python scripts/capture_dashboard.py
```

采集后先做 session 质量检查：

```bash
python scripts/validate_session.py --session-dir data/raw/session_YYYYMMDD_HHMMSS
```

如果要进一步检查 OpenVINS 链路，再按 `docs/system_workflow.md` 里的 OpenVINS 离线流程准备 `C0 + head_imu` 或对应测试相机和 IMU 数据。

## 5. 评分标准

每个候选位置按 1-5 分打分。`5` 表示非常好，`1` 表示不可用。

| 维度 | 5 分 | 3 分 | 1 分 |
| --- | --- | --- | --- |
| 手部可见性 | 指尖、关键关节、接触点长期清晰可见 | 部分动作可见，复杂动作有遮挡 | 关键动作经常看不到 |
| 爪夹/接触可见性 | 爪夹末端、开合状态、接触点都可见 | 只能看到部分结构或接触瞬间 | 无法判断夹取状态 |
| VIO 背景 | 稳定背景和纹理长期存在 | 背景偶尔存在，手部常占画面 | 画面几乎全是手/物体 |
| 图像质量 | 清晰、少模糊、曝光稳定 | 偶发模糊或失焦 | 高频失焦、过曝、模糊 |
| 机械可行性 | 不挡手、不碰撞、线缆容易固定 | 有轻微干涉，可改结构解决 | 明显影响抓握或易碰撞 |
| 同步/IMU | IMU 刚性固定，时间戳稳定 | IMU 可记录但刚性/同步需改进 | 无法满足 VIO 所需刚性或同步 |

推荐结论规则：

| 结论 | 条件 |
| --- | --- |
| 推荐安装位 | 手部可见性 >= 4，机械可行性 >= 4；若声明 VIO，则 VIO 背景 >= 4 且同步/IMU >= 4 |
| 备选安装位 | 手部可见性 >= 3，机械可行性 >= 3；存在明确结构修改路径 |
| 仅手部观察位 | 手部可见性 >= 4，但 VIO 背景 < 4 或 IMU 无法刚性固定 |
| 不推荐 | 手部可见性 <= 2，或机械可行性 <= 2，或会影响正常动作 |

## 6. 输出给机械/硬件团队的内容

最终需求文档应包含：

- 爪夹场景首选位置、备选位置、禁用位置。
- 普通精细动作场景首选位置、备选位置、禁用位置。
- 每个推荐位置的关键尺寸范围：高度、角度、横向偏移、相机朝向。
- IMU 放置要求：是否必须、是否与摄像头同刚体、频率、同步、标定要求。
- 单摄像头同时做手部特写和 VIO 的风险说明。
- 需要硬件预留的结构：可调支架、IMU 固定位、走线、标定定位面、拆装后复现基准。
- 每个候选位的测试证据：session 路径、截图、短视频、评分表和结论。
